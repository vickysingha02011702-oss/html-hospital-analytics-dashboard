## 📖 Overview

Meridian Health Analytics Platform is a browser-based Business Intelligence application designed to demonstrate how a well-structured dimensional data model can be transformed into interactive executive dashboards, business documentation, and actionable insights.

Although this project uses a healthcare dataset as its demonstration domain, the underlying architecture is intentionally designed to be reusable across multiple industries, making it a foundation for future analytics solutions beyond healthcare.

Unlike traditional dashboards that focus only on visualization, this platform combines **data exploration**, **business intelligence**, **documentation**, and **report generation** into a single, self-contained application that runs entirely in the browser without requiring a backend server or database.

# 🏗 Application Architecture

Meridian Health Analytics Platform follows a modular architecture where each component performs a dedicated responsibility within the analytics workflow.

Rather than tightly coupling data processing, visualization, and documentation, the application separates each concern into independent modules.

```text
                User
                  │
                  ▼
        Dataset Upload / Demo
                  │
                  ▼
        Dataset Processing Layer
                  │
                  ▼
        Project State Manager
                  │
      ┌───────────┼───────────┐
      ▼           ▼           ▼
 Dashboard    Documentation   Reports
      │           │           │
      └───────────┼───────────┘
                  ▼
           Browser Interface
```

This modular design improves maintainability and allows future enhancements without requiring major architectural changes.

# 🗂 Data Model

The healthcare demonstration project is built upon a dimensional data model consisting of one central fact table connected to multiple dimension tables.

The dimensional approach simplifies analytical queries while supporting interactive dashboard filtering.

## Fact Table

- Encounter Facts

## Dimension Tables

- Patient
- Date
- Hospital
- Department
- Consultant
- Diagnosis
- Procedure
- Payer
- Location

This star schema design enables efficient aggregation, slicing, filtering, and KPI generation across multiple business domains.

---

# ⭐ Star Schema

```text
                   Dim Date
                      │
                      │
Dim Patient ──────────┤
                      │
Dim Hospital ─────────┤
                      │
Dim Department ───────┤
                      │
Dim Consultant ───────┤
                      │
                Fact Encounter
                      │
Dim Diagnosis ────────┤
                      │
Dim Procedure ────────┤
                      │
Dim Payer ────────────┤
                      │
Dim Location ─────────┘
```

The star schema provides a clear separation between measurable business events and descriptive business entities, making it ideal for Business Intelligence and dashboard analytics.
